题目：石子游戏胜负判定（海信集团 26 届 Java 卷）

题目描述
有 n 堆石子，第 i 堆有 a_i 颗。小 C 和小 Z 轮流操作，每次可以从一堆取走至少 1 颗，无法操作者负。游戏开始前，小 C 会恰好进行 m 次修改：第 j 次给第 p_j 堆增加 k_j 颗。你需要在初始状态以及每次修改后判断，若小 C 作为先手，能否获胜。

输入格式
第一行两个整数 n m（1 ≤ n, m ≤ 2×10^5），表示堆数和修改次数。
第二行 n 个整数 a_1 … a_n（0 ≤ a_i ≤ 10^9），初始每堆石子数。
接下来 m 行，每行两个整数 p_j k_j（1 ≤ p_j ≤ n，0 ≤ k_j ≤ 10^9），表示在第 p_j 堆增加 k_j 颗。

输出格式
共 m+1 行：
第 1 行是初始结果；
接下来 m 行对应每次修改后的结果。
若小 C 获胜输出 win，否则输出 lose。

示例 1
输入
2 2
1 2
1 1
2 1
输出
win
lose
win

示例 2
输入
3 1
1 2 3
2 1
输出
lose
win

提示
经典 Nim 游戏，胜负由各堆石子数的异或和决定：异或和非零则先手必胜，否则必败。
每次修改只需对对应堆执行加法，然后重新计算全局异或和即可。
总复杂度 O(n + m)。
